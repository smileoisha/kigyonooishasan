# スキル：顧客向け提案書HTML生成 (/teiansho)

## 使い方
`/teiansho [指示書MDのパス]` で起動。
指示書パスが省略された場合は AskUserQuestion で確認する。

## 目的
顧客ごとの提案書HTMLを、指示書MDに基づいて1ページ1ファイルで生成する。

## 事前確認（スキル起動直後に実施）

以下の情報を引数またはAskUserQuestionで収集する：
- **指示書MDのパス**（必須）
- **顧客名**（指示書に記載がない場合のみ確認）
- **顧客フォルダ名**（`H:\共有ドライブ\40_顧客情報\` 直下のフォルダ名、指示書に記載がない場合のみ確認）

## 手順

### STEP 1：指示書を読む
指定された指示書MDファイルを Read で読み込み、以下を把握する：
- ページ構成（何ページ・各ページの内容）
- プラン金額・サービス内容
- 費用比較の比較対象・金額（省略可）
- ロードマップの期間・フェーズ・カテゴリ・内容
- 卒業についてのページ内容（該当する場合）
- 前提条件の項目と本文
- フッターの免責文・年月

### STEP 2：リファレンスを読む
以下を Read する：

1. `C:\Users\kikuchi yuki\.claude\projects\H---------60-Web----kigyonooishasan\memory\reference_proposal_html.md`
   → ページ構成・CSS設計・UIパターン・差し替えチェックリストを確認

2. ナレッジ「提案書HTML制作の落とし穴と対処法」（man_1779329249369_imax）
   → 設計原則・禁止事項・PDF化手順を確認

### STEP 3：ひな型を読む
`H:\共有ドライブ\40_顧客情報\神成裕輝\60_会社側のみ閲覧\` 配下の
個別ページHTMLをリファレンスとして Read する：
- `神成さん_提案書_0_表紙.html`（表紙：ブランド名・メインタイトル・宛名・日付・目次）
- `神成さん_提案書_1_プラン比較.html`
- `神成さん_提案書_2_設計フェーズ.html`
- `神成さん_提案書_3_ロードマップ.html`
- `神成さん_提案書_4_費用比較.html`
- `神成さん_提案書_5_卒業について.html`
- `神成さん_提案書_6_前提条件.html`

**表紙ページの設計ポイント（0_表紙）：**
- ヘッダー・フッターなし。上下にゴールドライン（下は逆グラデーション）のみ
- 中央配置のフレックスレイアウト（`justify-content: center; align-items: center`）
- 宛名はゴールドの短いラインで上下を挟む
- 下部に目次をゴールドパネルで横並び（`cover-toc` / `cover-toc-item`）
- ページ番号は指示書の構成に合わせて差し替える

### STEP 4：提案書HTMLを1ページ1ファイルで生成する

**ファイル命名規則：**
`{顧客名}_提案書_{番号}_{ページ名}.html`

**CSS設計（全ファイル共通）：**

```css
body {
  font-family: 'Noto Sans CJK JP', 'Noto Sans JP', sans-serif;
  background: #e8edf3;
  padding: 24px;
}

.page {
  width: 297mm;
  /* heightは指定しない（コンテンツに合わせて自動） */
  display: flex;
  flex-direction: column;
  background: #ffffff;
  box-shadow: 0 6px 32px rgba(0,0,0,0.18);
  margin: 0 auto;
}
```

**差し替えチェックリスト（各ファイルごと）：**
- [ ] `<title>` タグの顧客名
- [ ] `header .subtitle`（顧客名・ページ名）
- [ ] 各ページ固有のコンテンツ（指示書MDに従う）
- [ ] `footer .fn`（免責文）
- [ ] `footer .fb` の年月

### 禁止事項（厳守）
- **JSによる動的サイズ調整（beforeprint/afterprint）は書かない**
- **@pageのsizeを動的に変更するJSは書かない**
- **1ファイルに複数ページを詰め込まない**
- **@media print で height を固定しない**
- **フォントにYu Gothic等を指定しない（Noto Sans CJK JP統一）**

### STEP 5：ファイルを保存する

保存先：`H:\共有ドライブ\40_顧客情報\{顧客フォルダ名}\60_会社側のみ閲覧\`

各ページを個別ファイルとして保存する。
保存先フォルダが存在しない場合は院長に確認してから保存する。

### STEP 6：完了報告

以下を報告する：
- 保存した各ファイルの絶対パス一覧
- 生成したページ数・各ページの内容一覧
- 差し替えた主な箇所の概要
- **「PDF化はClaude.aiチャット側でHTMLファイルを渡して実行してください」と案内**

## PDF化について
PDF化はこのスキルの範囲外。
Claude.aiチャット側でPlaywrightスクリーンショット方式で実行する。
院長がHTMLファイルをClaude.aiに渡せば、Noto Sans CJK JP + 3倍解像度 + 300dpiでPDF生成される。
