# token-economy.md

**目的**: Claude Code セッションのトークン消費を抑えつつ、出力精度を維持する。
**使い方**: セッション冒頭で `@shared/token-economy.md を遵守してください` と宣言、または各カスタムコマンドから参照する。

---

## 1. セッション運用ルール（Claudeが守る行動規範）

### 1-1. 開始時の動作
1. タスクを受け取ったら、**実装前に作業計画を箇条書きで提示**し、院長の承認を待つ。
2. 読み込む必要があるファイルを**先に宣言**する(例: 「`client-lookup.md` と `service-menu.md` を参照します」)。
3. 宣言外のファイルを勝手に読まない・触らない。

### 1-2. 応答フォーマット
- **結論 → 根拠 → 次手順** の順で構成する。
- 前置き・謝辞・自己言及(「承知しました」「お任せください」等)は書かない。
- 不明点は**推測せず、1問だけ**質問する。複数質問は禁止。
- コード提示時は変更差分のみ。周辺の未変更コードを再掲しない。

### 1-3. ツール呼び出しの節約
- 同じファイルを1セッション中に複数回 `Read` しない(1回読んだら記憶を使う)。
- 探索的な `Grep` / `Glob` は**目的を1行で宣言してから**実行する。
- Bash は1コマンドで済む処理を分割しない。

### 1-4. 禁止事項
- 聞かれていないリファクタ提案。
- 「ついでに〇〇も直しました」系の範囲外改変。
- 長大なサンプルコード・架空のユースケース生成。
- 「他にお手伝いできることは〜」系の締め文句。

---

## 2. 院長側の運用チェックリスト

### セッション開始前
- [ ] タスクのゴールと完了条件を1行で言語化したか
- [ ] 関連ファイルを `@` で明示指定する準備があるか
- [ ] 前セッションの続きなら `/compact` で要約済みか

### セッション中
- [ ] 試行錯誤が3往復超えたら、一度立ち止まって設計を見直す
- [ ] 話題が変わるときは `/clear` か新セッション
- [ ] `shared/` 更新など文脈継続が重要な作業は同セッションで完結させる

### セッション終了時
- [ ] 次セッションに引き継ぐ情報は `CLAUDE.md` か `shared/` に記録
- [ ] 一時的な試行錯誤ログは残さない

---

## 3. 使い分けフローチャート

```
新しいタスク?
├─ YES → /clear または新セッション
└─ NO → 同じ文脈の続き?
         ├─ 履歴が肥大化 → /compact で圧縮
         └─ まだ軽い → そのまま継続
```

| 状況 | アクション |
|---|---|
| カスタムコマンド単発実行 | そのまま1セッション使い切り |
| 関連ファイル群をまとめて編集 | 同セッション継続(文脈維持が効く) |
| 別タスクに移る | `/clear` |
| デバッグで往復が長くなった | `/compact` |
| 明日以降に持ち越す | `CLAUDE.md` に決定事項を追記して終了 |

## 4. 指示テンプレート(コピペ用)

### 4-1. 汎用オープニング
```
@shared/token-economy.md を遵守してください。

【タスク】
(1行でゴールを記述)

【完了条件】
- (条件1)
- (条件2)

【参照ファイル】
- (必要なファイルのみ明示)

まず作業計画を提示してください。承認後に実装に入ります。
```

### 4-2. 実装タスク用
```
@shared/token-economy.md 遵守。

【実装したいこと】
(何を・どこに・なぜ)

【制約】
- 既存ファイルの構造は変更しない
- (その他の制約)

手順を3ステップ以内で提示 → 承認 → 実装の順で進めてください。
```

### 4-3. デバッグ・調査用
```
@shared/token-economy.md 遵守。

【症状】
(再現手順と期待動作・実際の動作)

【調査範囲】
- (このファイルのみ / この関数周辺のみ など限定する)

原因仮説を最大3つ挙げ、優先順位をつけて提示してください。
原因特定後に修正案を提案 → 承認 → 実装。
```

### 4-4. レビュー・相談用
```
@shared/token-economy.md 遵守。

【相談内容】
(設計判断・方針の是非など)

結論を先に、根拠を箇条書きで。
追加情報が必要なら1問だけ質問してください。
実装は不要、議論のみ。
```

## 5. トラブル時のリカバリ

| 症状 | 対処 |
|---|---|
| Claudeが先走って勝手に実装し始めた | 「ストップ。token-economy.md 1-1 に従って計画提示から」と指示 |
| 応答が冗長化してきた | 「以降、結論→根拠→次手順のみ。前置き禁止」と再指示 |
| 同じファイルを何度も読んでいる | 「既読ファイルの再読禁止。記憶から応答」 |
| 範囲外の改変が発生した | 「指示範囲外は差し戻し。最小差分で再提示」 |
